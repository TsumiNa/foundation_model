**Key design fact:** the `fm` CLI trains on target columns **as-is** — it does NOT normalize
targets (verified: `dataset.py` reads `data_column` directly; `[[tasks]].scaler` is inverse-only
for reporting). qc columns are pre-normalized in the parquet; the 7 non-qc regression columns are
not, so `prep_nonqc_norm.py` pre-normalizes them exactly like the deleted legacy script did:
`log1p(clip(x,>=0)) -> z-score(train-split stats) -> clip(+-5)`, writing `<col> (normalized)` +
a deterministic seed-42 `split` column into the `_norm.parquet` files. **To regenerate the derived
data on a fresh machine:** `cd /home/ea0094/projects/foundation_model && .venv/bin/python <path>/prep_nonqc_norm.py`
(the derived files already exist on rikyu).

`material_type` is trained with the **raw 5 labels** here (`num_classes=5`). The reference merged
them 5->3 via `{0:0,2:0,1:1,3:1,4:2}`; we did not, so material_type accuracy isn't 1:1 with the
reference (all other tasks are directly comparable).

## 3. The 8 configs + job scripts (in this dir)

- `configs/full24_baseline_0p05.toml` — baseline `default_replay=0.05` **with** `per_task=0.10` on
  the fixed tail (formation_energy, magnetic_moment, tc, klat, material_type). This mirrors the
  reference (`replay_ratio`/`replay_ratio_high`). Run by `jobs/rehearsal_full24.sbatch`.
- `configs/sweep_<tag>.toml` — 7 sweep variants, identical to the baseline **except** `[pretrain.
  rehearsal]` = `interval=1`, `default_replay=<value>`, **no `per_task`** (uniform across all old
  tasks, for a clean sweep). Run by `jobs/replay_sweep.sbatch` (selects the config via the
  `SWEEP_CFG` env var).

| tag | default_replay | meaning |
|---|---|---|
| baseline | 0.05 (+tail 0.10) | reference-faithful |
| 0p10 / 0p15 / 0p20 | 0.10 / 0.15 / 0.20 | fraction of each task's valid train labels |
| n100 / n200 / n500 / n1000 | 100 / 200 / 500 / 1000 | absolute label count per task |

`default_replay` is a float in (0,1) = fraction, or int >= 1 = absolute count. Count is converted
per task by `replay_to_ratio` = `min(1.0, count / n_valid_train)` — so **a count larger than a
task's valid train labels clamps to 100%** (full replay, no oversampling). See §5.

Each config writes to its own `[output].dir`:
- baseline -> `artifacts/pretrain_rehearsal_full24_rikyu/`
- sweep    -> `artifacts/replay_sweep/replay_<tag>_rikyu/`
(with `n_runs=1`, outputs land directly under `<dir>/training/` — metrics at
`<dir>/training/metrics_table.csv`, long format.)

**On rikyu these files live at** `/home/ea0094/jobs/rehearsal/` (baseline toml + `rehearsal_full24.sbatch`)
and `/home/ea0094/jobs/rehearsal/sweep/` (7 sweep tomls + `replay_sweep.sbatch`). The copies in
this repo dir are the source of truth.

stress retention of different tasks.

## 6. Final result (all 8 original runs)

MEAN reg/kr R² is the retention indicator (higher = less forgetting). Full per-task table:
`python analysis/compare_sweep.py`.

```
                  ref   0.05   0.10   0.15   0.20    100    200    500   1000
MEAN reg/kr R2   0.411  0.447  0.518  0.516  0.553  0.371  0.420  0.498  0.556
material_type ac 0.981  0.972  0.970  0.985  0.976  0.941  0.952  0.980  0.984
```

- **Reproduction is faithful**: 0.05 baseline mean R² 0.447 vs reference 0.411; per-task close.
- **More replay → better retention** in both families (fixed: 0.371→0.420→0.498→0.556 for
  100→200→500→1000; fraction: 0.447→0.518→…→0.553 for 0.05→0.10→…→0.20).
- **Saturation** (`analysis/saturation.py`, dense-task view, unified x = replay samples per dense
  qc task): fit `gap = 0.185/(1 + n/1765)`, no-forgetting ceiling ≈ 0.751. Knee (half-gap) ≈ n≈1765;
  **90% saturation ≈ n≈15,900/task, 95% ≈ n≈33,500/task** — approaching the ceiling needs replaying
  a large fraction of each dense task (expensive). Plot: `analysis/replay_saturation.png`.
  The relay jobs (n=1500/2000/2500) sit right around the knee to firm up that region.
- Sparse tasks flip: e.g. magnetic_susceptibility does better under small *fixed* counts (which
  give it its full 58 samples) than under small *fractions* (which give it ~3).
